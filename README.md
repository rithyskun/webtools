## Prompt Template API

A simple API-only project exposing a prompt template JSON object.

- **GET** `/api/prompt-template` – retrieves a default template. You can modify `src/app/api/prompt-template/route.ts` to generate dynamic templates.

Example response:

```json
{
  "title": "Default Prompt",
  "description": "A simple prompt template generated by the Next.js API.",
  "variables": [
    { "name": "userName", "type": "string", "example": "Alice" },
    { "name": "goal", "type": "string", "example": "Write a short story" }
  ],
  "template": "Hello {{userName}}, please help me {{goal}}."
}
```

This project includes a set of reusable libraries and middleware designed to harden the API:

- `src/lib/securityHeader.ts` – exports helpers like `addSecurityHeaders` (CSP, HSTS, etc.), `addNoCacheHeaders`, and CORS utilities.
- `src/lib/rateLimiter.ts` – provides a `RateLimiter` class with a few presets (`globalRateLimiter`, `webhookRateLimiter`, etc.) for tracking request counts.
- `src/lib/logger.ts` – advanced `Logger` class instance (exported as `logger`) that writes to daily files and exposes `logRequest` for structured API logging.
- `src/lib/requestLogger.ts` – convenience wrapper that measures duration and user-agent, calling `logger.logRequest`.
- `src/lib/inputSanitizer.ts` – class with static methods (`sanitizeString`, `sanitizeEmail`, etc.) and constants for file validations.
- `src/lib/apiKeyValidator.ts` – `ApiKeyValidator` class instantiated as `apiKeyValidator`; supports multiple keys and hashing.

Middleware (`src/middleware.ts`) now imports these updated modules, applies rate-limiting, API-key checks, logs requests via `requestLogger`, and adds security headers.

### Environment variables

- `API_KEYS` – comma‑separated list of allowed API keys. If omitted, all keys are considered valid (useful in development). Example:
  ```bash
  API_KEYS=key1,key2,anothersecret
  ```
  Set this in your local `.env` or via the Vercel dashboard (e.g. as `api_keys`).

Note: earlier versions used a single `API_KEY`; the new validator supports multiple values and also offers hashing.

### Vercel deployment

A `vercel.json` file is included for easy deployment:

```json
{
  "version":3,
  "builds":[{"src":"package.json","use":"@vercel/next"}],
  "env":{"API_KEY":"@api_key"},
  "routes":[{"src":"/api/(.*)","dest":"/api/$1"}]
}
```

On Vercel, add an environment variable named `api_key` (or directly `API_KEY`) to match the local value.

```bash
# local development
export API_KEY=supersecret
npm run dev
```
You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.
