## Getting Started

First install runtime dependencies (the examples below show `npm`, adapt if using yarn/pnpm/bun).
The `next-auth` dependency is specified with a wildcard version in `package.json`; choose a compatible release when you have internet access.

```bash
npm install
# you may need to run this on a machine with registry access if the container is offline
npm install next-auth            # (omit version or pick e.g. "next-auth@^4")
```

Then start the development server:

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
# or
bun dev
```

This project is intended to be used as an API service; there is no public frontend. Open [http://localhost:3000/api/prompt-template](http://localhost:3000/api/prompt-template) to access the prompt template endpoint.


## Prompt Template API

A simple API-only project exposing a prompt template JSON object.

- **GET** `/api/prompt-template` ŌĆō retrieves a default template. You can modify `src/app/api/prompt-template/route.ts` to generate dynamic templates.

Example response:

```json
{
  "title": "Default Prompt",
  "description": "A simple prompt template generated by the Next.js API.",
  "variables": [
    { "name": "userName", "type": "string", "example": "Alice" },
    { "name": "goal", "type": "string", "example": "Write a short story" }
  ],
  "template": "Hello {{userName}}, please help me {{goal}}."
}
```

This project includes a set of reusable libraries and middleware designed to harden the API:

- `src/lib/securityHeader.ts` ŌĆō exports helpers like `addSecurityHeaders` (CSP, HSTS, etc.), `addNoCacheHeaders`, and CORS utilities.
- `src/lib/rateLimiter.ts` ŌĆō provides a `RateLimiter` class with a few presets (`globalRateLimiter`, `webhookRateLimiter`, etc.) for tracking request counts.
- `src/lib/logger.ts` ŌĆō advanced `Logger` class instance (exported as `logger`) that writes to daily files and exposes `logRequest` for structured API logging.
- `src/lib/requestLogger.ts` ŌĆō convenience wrapper that measures duration and user-agent, calling `logger.logRequest`.
- `src/lib/inputSanitizer.ts` ŌĆō class with static methods (`sanitizeString`, `sanitizeEmail`, etc.) and constants for file validations.
- `src/lib/apiKeyValidator.ts` ŌĆō `ApiKeyValidator` class instantiated as `apiKeyValidator`; supports multiple keys and hashing.

Middleware (`src/middleware.ts`) now imports these updated modules, applies rate-limiting, API-key checks, logs requests via `requestLogger`, and adds security headers.

### Environment variables

- `API_KEYS` ŌĆō commaŌĆæseparated list of allowed API keys. If omitted, all keys are considered valid (useful in development). Example:
  ```bash
  API_KEYS=key1,key2,anothersecret
  ```

Note: earlier versions used a single `API_KEY`; the new validator supports multiple values and also offers hashing.

#### Single signŌĆæon (SSO)

This project can be configured to accept OAuth logins via [NextAuth.js](https://next-auth.js.org).
You can enable any combination of Auth0, GitHub and Google (Gmail) accounts.
The authentication routes are exposed under `/api/auth/*` and protected helpers are
available in `src/lib/auth.ts`.

Required environment variables (see `.env.example` for placeholders):

```bash
NEXTAUTH_SECRET=some-random-value

# Auth0
AUTH0_CLIENT_ID=ŌĆ”
AUTH0_CLIENT_SECRET=ŌĆ”
AUTH0_ISSUER=https://your-tenant.auth0.com

# GitHub
GITHUB_CLIENT_ID=ŌĆ”
GITHUB_CLIENT_SECRET=ŌĆ”

# Google
GOOGLE_CLIENT_ID=ŌĆ”
GOOGLE_CLIENT_SECRET=ŌĆ”
```

Once the variables are set, start the dev server and visit `http://localhost:3000/api/auth/signin`.
You can protect API endpoints by calling `requireAuth(req)` from `src/lib/auth.ts`;
`src/app/api/prompt-template/route.ts` contains an example of a guarded route.

On Vercel simply add the corresponding environment variables in the dashboard.


### Vercel deployment

A `vercel.json` file is included for easy deployment:

```json
{
  "version":3,
  "builds":[{"src":"package.json","use":"@vercel/next"}],
  "env":{"API_KEY":"@api_key"},
  "routes":[{"src":"/api/(.*)","dest":"/api/$1"}]
}
```

On Vercel, add an environment variable named `api_key` (or directly `API_KEY`) to match the local value.

```bash
# local development
export API_KEY=supersecret
npm run dev
```
You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.
